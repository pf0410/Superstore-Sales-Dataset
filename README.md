零售销售数据分析与可视化仪表盘
项目描述

本项目基于 Superstore Sales Dataset，对零售业务的销售表现与盈利结构进行探索性数据分析，并构建交互式数据可视化仪表盘。

数据集来源：
https://www.kaggle.com/datasets/rohitsahoo/sales-forecasting/versions/1/data

项目使用 Python（Pandas）进行数据清洗与预处理，通过 MySQL 实现数据存储与中转，最终利用 Power BI 构建数据分析仪表盘，从销售趋势、产品结构和客户贡献度等多个维度分析企业经营表现。

分析重点包括：
销售收入时间趋势分析
产品类别盈利结构分析
客户销售贡献度分析
产品子类别销售效率分析
通过数据可视化与指标分析，识别业务增长驱动因素及潜在盈利风险。

技术实现
数据清洗：Python（Pandas）
数据存储：MySQL
数据分析与可视化：Power BI

关键分析发现
明显的季节性销售模式
    销售收入在第四季度（Q4）达到峰值，显示企业销售具有明显的节假日需求特征。
科技类产品是主要增长引擎
    Technology 类产品贡献了最高销售额且利润率较高，是企业收入增长的重要来源。
存在结构性盈利问题
    部分子类别（如 Tables）虽然销售额较高，但整体利润为负，反映出潜在的定价或成本控制问题。
收入集中度较高
    少数客户贡献了较大比例销售额，企业在客户结构上存在一定集中度风险。
部分高销售客户利润为负
    部分客户销售额较高但整体利润为负，说明部分订单可能存在较高折扣或利润空间不足的问题。

项目成果
使用 Python（Pandas）完成数据清洗与预处理，处理缺失值、重复数据并标准化字段格式。
通过 MySQL 构建结构化数据表，实现数据存储与分析流程衔接。
使用 Power BI 构建交互式销售分析仪表盘，包括销售趋势分析、产品利润结构分析、客户贡献度分析等核心报表。
通过 Pareto分析识别核心客户群体，并发现部分高销售客户利润为负的潜在业务问题。
